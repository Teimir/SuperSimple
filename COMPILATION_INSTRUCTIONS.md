# Инструкции по компиляции LaTeX документации

Этот документ содержит инструкции по компиляции LaTeX документации проекта в PDF формат.

## Файл документации

Основной файл LaTeX документации находится в:
- `doc/documentation.tex` - Полная документация проекта на русском языке

## Метод 1: Использование Overleaf (Рекомендуется)

Overleaf - это онлайн-редактор LaTeX, который не требует установки дополнительного программного обеспечения.

### Шаги:

1. Перейдите на [Overleaf](https://www.overleaf.com/)
2. Создайте новый проект или загрузите существующий
3. Загрузите файл `doc/documentation.tex` в проект
4. Если в документе используются дополнительные файлы (изображения, стили), загрузите их тоже
5. Нажмите кнопку "Recompile" или "Compile"
6. PDF будет автоматически сгенерирован и доступен для скачивания

### Преимущества:
- Не требует установки LaTeX дистрибутива
- Автоматическая установка пакетов
- Совместная работа над документом
- История версий

## Метод 2: Локальная компиляция

Для локальной компиляции требуется установленный LaTeX дистрибутив.

### Требования

#### Windows:
- [MiKTeX](https://miktex.org/) или [TeX Live](https://www.tug.org/texlive/)

#### Linux:
```bash
# Ubuntu/Debian
sudo apt-get install texlive-full

# Fedora
sudo dnf install texlive-scheme-full
```

#### macOS:
- [MacTeX](https://www.tug.org/mactex/)

### Компиляция

#### Использование pdflatex:

```bash
cd doc
pdflatex documentation.tex
pdflatex documentation.tex  # Повторная компиляция для правильных ссылок
```

#### Использование xelatex (если нужна поддержка Unicode):

```bash
cd doc
xelatex documentation.tex
xelatex documentation.tex
```

#### Использование lualatex:

```bash
cd doc
lualatex documentation.tex
lualatex documentation.tex
```

### Необходимые пакеты

Документация может использовать следующие стандартные пакеты LaTeX:
- `geometry` - для настройки полей страницы
- `hyperref` - для гиперссылок
- `listings` - для листингов кода
- `graphicx` - для изображений
- `amsmath` - для математических формул
- И другие стандартные пакеты

Большинство современных LaTeX дистрибутивов включают эти пакеты по умолчанию.

## Метод 3: Использование Makefile (если доступен)

Если в проекте есть Makefile для компиляции документации:

```bash
cd doc
make documentation.pdf
```

## Решение проблем

### Ошибка "Package not found"

Если компилятор сообщает об отсутствии пакета:

**MiKTeX (Windows):**
- Пакеты устанавливаются автоматически при первом использовании
- Или используйте MiKTeX Package Manager

**TeX Live:**
```bash
# Установка конкретного пакета
tlmgr install <package_name>
```

**Overleaf:**
- Пакеты обычно устанавливаются автоматически
- Если проблема сохраняется, проверьте синтаксис `\usepackage`

### Ошибки кодировки

Если возникают проблемы с кириллицей:
- Убедитесь, что файл сохранен в UTF-8
- Используйте `xelatex` или `lualatex` вместо `pdflatex`
- Проверьте наличие пакета `babel` или `polyglossia` для русского языка

### Проблемы с путями

Если используются относительные пути к файлам:
- Убедитесь, что вы запускаете компиляцию из правильной директории
- Проверьте, что все необходимые файлы находятся в указанных путях
